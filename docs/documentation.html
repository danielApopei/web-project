1. Introducere

DeAd (Detention Admin) este o aplicație web pentru gestionarea vizitelor persoanelor condamnate la executarea unei pedepse într-un penitenciar sau într-o casă de corecție.

2. Funcționalități principale

  • Înregistrarea vizitelor, inclusiv informații despre vizitatori și natura vizitelor.
  • Consemnarea datelor despre intrevedere, inclusiv data, durata, obiecte/date furnizate, rezumatul discuțiilor.
  • Generarea de statistici privind vizitele, inclusiv documente HTML, CSV și JSON.
  • Oferea funcționalităților esențiale prin intermediul unui API Web REST/GraphQL.

3. Structura de bază a paginii

Fiecare pagină HTML a aplicației are o structură de bază comună:

<!DOCTYPE html>
<html>
<head>
<!-- Metadate și legături către fișierele CSS și JavaScript -->
</head>
<body>
<!-- Conținutul vizibil al paginii -->
</body>
</html>

4. Elemente HTML specifice

4.1. Formular de înregistrare a vizitelor

Pentru înregistrarea unei noi vizite:

<form action="/register-visit" method="post">
<label for="visitor-name">Numele vizitatorului:</label>
<input type="text" id="visitor-name" name="visitor-name">
<!-- Alte câmpuri pentru informații despre vizitator și natura vizitei -->
<button type="submit">Înregistrare vizită</button>
</form>

4.2. Tabel pentru afișarea vizitelor

Pentru afișarea vizitelor într-un tabel:

<table>
<thead>
<tr>
<th>Data vizitei</th>
<th>Durata</th>
<!-- Alte anteturi pentru informații despre vizită -->
</tr>
</thead>
<tbody>
<tr>
<td>12-04-2024</td>
<td>1 oră</td>
<!-- Alte celule pentru informații despre vizită -->
</tr>
<!-- Alte rânduri pentru fiecare vizită -->
</tbody>
</table>
